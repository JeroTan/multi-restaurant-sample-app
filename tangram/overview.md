# Project Overview: Multi-Restaurant SaaS QR Ordering Platform

## 🎯 Core Vision
A multi-tenant SaaS platform enabling restaurants to digitize their ordering process through unique table-based QR codes. Customers order via mobile browsers (no app required), while staff manage real-time orders through a unified dashboard.

## 🏗️ Technical Pillars
1.  **Architecture:** Next.js 14+ (App Router) for a full-stack, unified codebase.
2.  **Multi-tenancy:** PostgreSQL with strict data isolation (Row-Level Security or tenant filtering).
3.  **Real-time:** Server-Sent Events (SSE) or WebSockets for instant staff notifications.
4.  **Auth:** NextAuth.js/JWT with Role-Based Access Control (Customer, Staff, Admin, Super Admin).
5.  **Storage:** Cloud Object Storage (S3/OSS) for menu assets with automatic compression.
6.  **I18n:** `next-intl` for multilingual customer interfaces (EN/CN).

## 🚀 Phase I: MVP Scope
- **Customer H5:** QR Scan -> Menu -> Cart -> Order Submission & Status.
- **Staff Dashboard:** Real-time order queue with status transitions.
- **Restaurant Admin:** Menu management (Categories, Dishes, Variants) and QR/Table generation.
- **Platform Admin:** Tenant onboarding and subscription management.

## 🗺️ Roadmap
- **Step 1: Preconstruct** - Deep dive into requirements, legacy constraints, and technical design.

- **Step 3: Feature Implementation** - Atomic tasks for H5, Dashboards, and Multi-tenancy.
